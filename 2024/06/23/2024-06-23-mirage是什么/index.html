mirage 是什么

By limeng32

发表于 2024-06-23

为什么要开发 mirage

目前有很多价值很高的 react 单页应用，但缺乏一个将它们集成在一起的完美方案。设想一下，如果有一种工具能将单页应用方便的打包为 vscode 插件，借助 acquireVsCodeApi 接口实现单页应用操作本地文件系统、shell 和发起网络请求，将极大提高这些单页应用的可用性，因此 mirage 应运而生。

mirage 的简单介绍

mirage 是一个打包插件的工具，目前已经发布了 mirage-lowcode（低代码设计器）、mirage-swagger（swagger 管理）、mirage-flow（流程图设计器），可以从 vscode 插件市场直接下载。一个由 mirage 打包的插件如下所示：

从上图可以看出，在插件中单页应用的表现形式和在浏览器中没有区别，但是借助于 acquireVsCodeApi 接口，单页应用可以获取更多的权限，比如读取/保存本地文件（不只限于 Downloads 目录）等等，在后面的文章中我们会进行介绍。

vscode 插件的优势

  • 可以在本地保存/读取数据，进行高频率读写；

  • 可以调用本地 shell，例如 git、npm、mvn；

  • 借助于统一的登录鉴权方案，可以将多个应用无缝集成，并在 vscode 层面实现这些应用互相通信；

  • 更方便的升级推送机制；

如何查看 mirage 的源代码

mirage 的代码在 gitee 上进行开源，地址见 https://gitee.com/limeng32/mirage_box，基于 monorepo 方式管理，所有相关内容都在同一仓库中的不同分支中，各分支含义如下：

  • master 放置开源协议和 README 文件，无具体开发内容

  • main 放置打包原始框架，打包新插件前应该从这里拉取新的分支

  • hexo 放置 mirage 说明文档源码的分支

  • feature/lowcode 放置打包 mirage-lowcode 插件的代码

  • feature/swagger 放置打包 mirage-swagger 插件的代码

  • feature/flow 放置打包 mirage-flow 插件的代码

  • feature/test 放置用来做打包练习的 mirage-test 插件的代码
